# CGT Carbon Farming

Cooperative Game Theory (CGT) framework for carbon credit market participation by smallholder farmers. The project models how groups of small farmers can form coalitions to jointly participate in voluntary carbon markets, optimise their sequestration practice portfolios, and distribute the resulting surplus fairly.

## Data

### `data/Dataset_INR/`

| File | Description |
|---|---|
| `15_farmers.csv` | Farmer profiles for the 15-farmer experiment: farmer ID, farm size (ha), per-ha budget (INR/season), and total budget (INR/season). |
| `25_farmers.csv` | Same structure as above, extended to 25 farmers. The first 15 rows are identical to `15_farmers.csv`. |
| `alpha_carbon.csv` | 20×20 symmetric matrix of pairwise **carbon sequestration interaction** coefficients (α_jk). Entry (j,k) is the additive bonus/penalty to total sequestration when practices j and k are adopted together. `-inf` entries denote incompatible practice pairs. |
| `Beta_cost.csv` | 20×20 symmetric matrix of pairwise **operational cost interaction** coefficients (β_jk). Entry (j,k) is the additive cost adjustment when practices j and k are adopted together. `-inf` entries denote incompatible pairs. |
| `gamma_yield.csv` | 20×20 symmetric matrix of pairwise **yield impact interaction** coefficients (γ_jk). Entry (j,k) is the additive adjustment to yield revenue loss/gain when practices j and k are co-adopted. |
| `incompatibility_inr.csv` | Explicit incompatibility constraints between practices. Documents which practice pairs cannot be co-adopted by a farmer. Cross-reference with `-inf` entries in the α and β matrices. |
| `master_file.csv` | Master lookup table for all 20 carbon sequestration practices. Contains per-practice baseline values: operational cost components (fixed and variable), baseline carbon sequestration potential (CSP, tCO₂/acre/year), and yield impact (INR/acre/year). |
| `revised_20_practice_interaction_matrices_v3.csv` | Source/combined file from which the individual α, β, and γ matrices were derived. Useful as a cross-reference if individual matrix entries appear inconsistent. |

### `data/processed/`

Preprocessed inputs serialised as Python pickle files. Generated by Notebook 01 and consumed by Notebooks 02–04.

| File | Description |
|---|---|
| `optimization_inputs_15_farmers.pkl` | All optimisation-ready inputs for the 15-farmer setting: farmer parameters, practice cost/sequestration/yield vectors, and interaction matrices loaded into a structured format. |
| `standalone_values_15_farmers.pkl` | Precomputed standalone values ṽ({i}) for each of the 15 farmers — the net profit each farmer achieves by participating in the carbon market independently. These serve as the individual rationality floor in the coalition and allocation stages. |

## Notebooks

Run in sequence. Each notebook depends on outputs from the previous one.

### `01_Data_Ingestion_and_Preprocessing.ipynb`
Loads raw CSVs from `Dataset_INR/`, validates and cleans the data (handles `-inf` incompatibility entries, checks matrix symmetry), and serialises the processed inputs to `data/processed/`. Start here if you are setting up the pipeline for the first time or adding new farmer/practice data.

### `02_Single_Farmer_Problem.ipynb`
Solves the standalone optimisation for each farmer independently. Given a farmer's budget, farm size, and the practice interaction matrices, it finds the optimal practice portfolio that maximises the farmer's net profit from carbon credit revenue, minus operational costs, yield impacts, and solo MRV/transaction costs. Outputs are the standalone values ṽ({i}), saved to `standalone_values_15_farmers.pkl`.

### `03_Coalition_Problem.ipynb`
Solves the joint portfolio optimisation for the grand coalition and every sub-coalition. For a given coalition S, it finds the optimal heterogeneous practice profile (each farmer may adopt a different portfolio) that maximises total coalition carbon revenue minus shared MRV and transaction costs. Three coalition feasibility variants are explored: pooled residual (CF1), area-weighted cost share (CF2), and sequestration-weighted cost share (CF3). Outputs are the characteristic function values v(T) for all subsets T.

### `04_Surplus_Sharing.ipynb`
Takes the characteristic function values from Notebook 03 and solves the allocation problem. Computes and compares five allocation mechanisms:
- **Least core** — minimises the maximum incentive to defect; most robust when the core is empty.
- **Nucleolus** — lexicographically minimises sub-coalition dissatisfaction; unique and always in the least core.
- **Shapley value** — allocates by expected marginal contribution; core membership verified post-hoc.
- **Equal split** — divides v(S) equally; baseline only.
- **Proportional split** — divides v(S) proportionally to each farmer's sequestration contribution; baseline only.

## Key Concepts

**Standalone value ṽ({i})**: The net profit farmer i achieves by participating in the carbon market alone, bearing the full MRV and transaction costs. This is the minimum any allocation rule must give farmer i (individual rationality floor).

**Coalition value v(S)**: The maximum net surplus coalition S can generate through joint portfolio optimisation and shared certification costs. Superadditive structure means v(S) ≥ Σ ṽ({i}) in most configurations, motivating coalition formation.

**Core**: The set of allocations where no sub-coalition has an incentive to defect. May be empty for non-convex games.

**Least core / Nucleolus**: Fallback allocation concepts used when the core is empty, providing the strongest available stability guarantees.

## Notes for Collaborators

- All monetary values are in **Indian Rupees (INR)** per season unless otherwise noted.
- Farm sizes are in **hectares (ha)**; carbon sequestration figures are in **tCO₂/acre/year** (note the unit mismatch — conversion is applied in preprocessing).
- The 20 practices covered are listed as column/row headers in the α, β, and γ matrices.
- `-inf` in any interaction matrix means the practice pair is **incompatible** and must not be co-adopted. This is enforced as a hard constraint in the optimisation.
- Notebooks 03 and 04 have exponential complexity in the number of farmers (O(2^n) sub-coalition solves). The 15-farmer setting is the practical limit for exact computation; larger farmer sets require approximation methods.
